Overview

This project leverages a local Large Language Model (LLM) that autonomously decides whether to retrieve documents from a local vector store or perform a web search. After generating an initial answer, the system performs both reflection and hallucination checks before finalizing the response.

Technical Workflow

1. Indicator & Retrieval

  • Indicator Function: Routes each question to either Chroma DB (local) or Web Search (external).
  • Retrieve/Search:
    • Chroma DB: Fetch local documents.
    • Web Search: Gather external data if local context is insufficient.

2. Generation & Reflection

  • Generate: LLM drafts an initial answer from retrieved context.
  • Reflect: Check for completeness and accuracy. If Reflection_Status is Incomplete, loop back to Generate.
  • Hallucination Check: Validate factual correctness; regenerate if inaccuracies are found.

3. Adaptability & Goal-Oriented Loop

A state graph monitors relevance and iteratively refines answers‚Äîlooping between Indicator, Retrieve/Search, and Generate/Reflect until a thorough, accurate response is produced.
