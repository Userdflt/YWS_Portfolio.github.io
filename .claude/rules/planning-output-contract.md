
# Planning Output Contract

When the task is planning or a plan document:

- `Relevant Rules and Skills` — list names + one-line "why" each. No paragraphs.
- `Module Map` — table or bullet list: file/module → what it owns. One line per entry.
- `Scope Contract` — `Audit Baseline`, `In-Scope`, `Non-Goals`, `Change-Trigger Policy` subsections. Bullets only. See `.claude/rules/scope-contract.md`.
- `Cleanup Plan` — list candidates as: `path/thing → removal gate`. One line each. If none expected, state "None" — don't omit the section.
- Prefer root-cause fixes over parallel old/new paths.
